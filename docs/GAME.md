# โมเดลเกม Fishbone Cast — สูตร ราคา และ DOM selector

> ทุกค่าแกะจากโค้ดเกมจริง · อัปเดตเมื่อเกมอัปเดตแล้ว selector/สูตรเปลี่ยน

## สูตรคะแนน (ใช้เล็งเข็ม)

| เฟส | เงื่อนไข | คะแนน | ค่าคงที่ในบอท |
|---|---|---|---|
| เกจตวัด (cast) | diff ≤ 0.09 | `100 − diff/0.09×40` → 60–100 | `CAST_SPAN=.09, CAST_DROP=40` |
| ดึงปลา (reel) | diff ≤ 0.0165 | 100 ทันที (โซนแดง) | — |
| ดึงปลา (reel) | diff ≤ 0.11 | `100 − diff/0.11×30` → 70–95 | `REEL_SPAN=.11, REEL_DROP=30` |

⚠️ reel ไม่มีคะแนน 96–99 จริง (กระโดด 95→100) — `scoreToOffset` เล็งกลางเมื่อ score>95

## สูตรผลลัพธ์ปลา

- `g = score/100`
- โอกาสขยะ: `junk = max(0.03, 0.30 − g×0.27)` — คะแนนสูง = ขยะน้อย
- ตัวคูณโชค: `m = g²×1.5 + baitBonus + rod + outfit + potion` — **ครอบงำโดย g** (ที่ 96–100 ≈ 1.44) · baitBonus แค่ 0.02→0.33
- ราคา: `price = base × (1 + 0.5×weightFrac)` · shiny ✨ = ×10
- เกม auto-fish ของเกมเองใช้คะแนนแย่โดยเจตนา (10–45) → บอทเรากำไร ~2 เท่า

**ข้อสรุปเศรษฐกิจ: เหยื่อถูกมักคุ้มกว่า** — เหยื่อแทบไม่เปลี่ยนคุณภาพปลา จ่ายแพงเปล่า (พิสูจน์จากข้อมูลจริง: ขั้น 3 กำไร +13.5k/แพ็ค vs ขั้น 7 ขาดทุน −9k/แพ็ค ที่ท่าเรือทะเล)

### 🎯 เข้าถึง Phaser game + ผลตก ผ่าน React state (v6.83 — กุญแจสำคัญ)

เกม = **Phaser 4 (WebGL)** ใน Next.js/React → เข้าถึง object ได้ (ไม่ต้องอ่าน pixel):
```js
// เดินจาก <canvas> ขึ้น React fiber หา hook ที่ค่าเป็น Phaser game (มี .scene/.renderer/.textures/.isBooted)
canvas → __reactFiber$ → เดิน .return ~2 ชั้น · เช็คทั้ง fiber + .alternate (React double-buffer) · re-resolve สดทุกครั้ง (fiber stale ได้)
```
- **Phaser game**: `game.scene.keys.main` = ฉากหลัก · `.sys.displayList.list` = game objects (x,y,texture.key,fillColor,text) · `scene.isFishing` = bool กำลังตก · `scene.bobber/player/mapManager/...` = state โลก
- **ผลตกแต่ละตัว** (สำคัญสุด): React hook ที่เป็น **array `{id, r:{fish,rarity,weight,isShiny,price,bait,mapId,baits,energy,level,rod,xp}}`** — โผล่ชั่วคราว ~1วิ ตอนโชว์ผลแล้วเคลียร์ · หาได้จากทรง (`v[0].r.fish`) ไม่ต้องพึ่ง index (minified เปลี่ยนได้)
- **catch-popup texture**: `catch-popup-<ชื่อปลา>` เป็น Image ในฉาก (ผลตกวาดบน canvas — จึงไม่มี popup DOM ในโหมด auto)
- บอทใช้ `readGameCatchArr()`+`pollGameCatches()`+`recordGameCatch()` อ่าน array นี้ (ทุกโหมดยกเว้น off) → **เก็บสถิติต่อตัว + Advisor ทำงานครบ** (ยืนยันสด: อ่าน "ปลาดุก" price54 weight2.45 bait2 ได้จริง)
- **v6.84: โหมด bot ตกเองได้ครบวงจรแล้ว** (เอนจินใหม่ · ดูตารางเฟสด้านบน) — ตกเองได้โบนัสโชค/คะแนนสูงกว่า auto เกม จึงเป็นค่าเริ่มต้น · gameauto = สำรอง
- **`gameState()` v6.82**: bite = `qBtn('ตกปลา (F)').textContent.includes('❗')` · result = ปุ่มขึ้นต้น "ตกต่อ!"/"เก็บปลา" · `pressSpace()` กด orb "ตกปลา (F)" ด้วย (แต่โหมด bot response ทำเฉพาะ fishMode==='bot')

**ระบบ "ตกปลาอัตโนมัติ" ของเกม** (โหมด `fishMode:'gameauto'` ค่าเริ่มต้น — จัดการครบทุกเฟสบน canvas เอง):
- ปุ่มเดียวสลับ `ตกปลาอัตโนมัติ` ⇄ `หยุดตกอัตโนมัติ` (aria-label ทั้งคู่) · helpers `gameAutoRunning()`/`startGameAuto()`/`stopGameAuto()`
- ยืนยันสด: start → running + ปุ่ม `ตกปลา (F)` **disabled** · stop → กลับ idle + enabled คืน
- **ข้อจำกัดสำคัญ (ยืนยันสด v6.82)**: auto ของเกม **ไม่โชว์ popup ผลใน DOM** (ตกจบบน canvas · เห็นแค่จำนวนเหยื่อลด) → `state==='result'` ไม่เคย fire → **เก็บสถิติต่อตัว/`pushCatch`/Advisor/ทดสอบเหยื่อ ในโหมด gameauto ไม่ได้** (ได้แค่ยอดขายรวมจาก sell toast)

**เทคนิคตรวจ DOM สด** (Claude for Chrome): globals ไม่ persist ข้าม `javascript_exec` (แต่ละ call = world แยก) · **DOM แชร์กัน** → เขียน log ลง `<div id="__fishlog">` / attribute แล้วอ่าน call ถัดไป · `setInterval` จาก call ก่อนยังรันพื้นหลังเขียน DOM ได้ · ปุ่มตอบด้วย fireClick (pointerdown+pointerup+click) · `outerHTML` โดน filter บล็อก ("[BLOCKED: Cookie]") → เก็บแค่ class/color/text

## ข้อจำกัดแพลตฟอร์ม

- ต้องมี `@grant` (unsafeWindow + GM_xmlhttpRequest) — `@grant none` โดน CSP ของ Vercel บล็อก
- Telegram ต้องผ่าน `GM_xmlhttpRequest` เท่านั้น (fetch โดน CSP)
- ยิง event ผ่าน `unsafeWindow` (W) ไม่งั้น React ไม่ได้ยิน
- เกมเด้ง "พักจอนาน 5 นาที" เมื่อ `document.hidden` นาน → บอท override เป็น false ตลอด (`keepTabAlive`)
