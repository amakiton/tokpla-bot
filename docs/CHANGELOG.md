## v6.142 — 🎯 แก้ต้นตอ "WASD เดินอัตโนมัติไม่ทำงาน" (window → document)

**บั๊กต้นเหตุที่หลอกมาทั้ง session:** บอทยิงปุ่มเดิน (KeyW/A/S/D) ไปที่ `W` (unsafeWindow = window) แต่**เกมฟัง keyboard event ที่ `document` ไม่ใช่ `window`** → เกมไม่ได้ยิน → ตัวละครไม่ขยับ → auto-travel / หนีถ้ำบอส / เดินไปหาบอส **พังหมดทุกฟีเจอร์ที่ต้องเดิน**
- **ทดสอบสด** (river_bank, ตัวเดียวกัน): ยิง `window.dispatchEvent` ตัวขยับ **12px** · ยิง `document.dispatchEvent` ขยับ **706px** · ยิง canvas (โฟกัสก่อน) 215px
- แก้: เพิ่ม `bossFireKey(type,c,k)` ยิง event ใหม่ไปทั้ง **document + canvas + window** (1 KeyboardEvent dispatch ได้ครั้งเดียว จึงสร้างใหม่ทุกเป้า) · `bossHold` เรียกผ่านตัวนี้
- **พิสูจน์แล้ว:** ใช้ document-dispatch พาตัวละครเดินเอง boss_cave → village → river_bank → **ข้ามสะพาน** → sea_dock สำเร็จ (เดินข้ามแมพจริง 3 ต่อ + เลี่ยงกำแพง/แม่น้ำ)
- **ต้อง deploy ขึ้น VPS:** บอทบน VPS ยังเป็น v6.141 (ยิง window) → auto-travel/ล่าบอสยังพังจนกว่าจะกด "ตรวจหาอัปเดต" เป็น v6.142
- **หมายเหตุ:** โหมดตกปลาปกติไม่กระทบ (ไม่ต้องเดิน) — บั๊กนี้กระทบเฉพาะฟีเจอร์ที่ต้องเดินข้ามที่/ข้ามแมพ

## v6.140 — 🐛 แก้ "บอทติดในถ้ำบอสถาวร" เมื่อเปิดล่าบอส (จาก healthcheck log จริง)

**อาการ (log สด 6 ชม.):** บอทเข้าถ้ำบอส → บอส "⌛ ไม่มาในเวลาที่รอ" → `👹 กลับบ้านไม่สำเร็จ (อยู่ boss_cave)` → วนซ้ำ + สแปม `ปุ่มตกปลากดไม่ได้ — เดินไปใกล้บ่อก่อน 🎣` ทุก ~5 วิ **ติดยาว 30+ นาที ฟาร์มศูนย์** (`bossHunt:true`, idle, ไม่มีบอส)
- **ต้นตอ:** `strandedInBossCave` (safety-net เดินออกถ้ำ + แจ้งเตือนถ้าเดินไม่ได้) ถูกล็อกด้วย `!isOn('bossHunt')` ตั้งแต่ v6.138 → พอ `bossFightHere` return กลับบ้านล้มเหลว (`bossTravelTo` คืน false) ขณะเปิดล่าบอส บอทติดในถ้ำถาวร: ตกปลาในถ้ำไม่ได้ + escapeBossCave ไม่มีวันได้รัน + ไม่มี alert
- **แก้:** ตัดเงื่อนไข `!isOn('bossHunt')` ออกจาก `strandedInBossCave` — ช่วง "เดินไป/สู้บอสจริง" ถูกกันด้วย `orchestrating` + `bossPhase!=='idle'` อยู่แล้ว, บอสโผล่จริงถูกกันด้วย `!present` (→ tick เรียก `bossFightHere` แทน) · escape จึงยิงเฉพาะสถานะ "idle + ในถ้ำ + ไม่มีบอส" = ติดจริงเสมอ · เดินออกได้ = ฟาร์มต่อ, เดินไม่ได้ (คุมตัวไม่ได้) = แจ้ง Telegram + backoff 5 นาที (escapeBossCave เดิม)
- **ไม่ผูก escape กับ boss timer โดยตั้งใจ** — เพราะบั๊กนี้เกิดตอน timer ว่า "บอสมา" แต่ `raidBossState().present` ไม่เคยเป็น true ("ไม่มาในเวลาที่รอ") → ถ้ากัน escape ตอน timer ใกล้ = สร้างกับดักเดิมซ้ำ
- **ผลรวม:** บอทไม่ติดถ้ำถาวรอีก — return ล้มเหลว = escape พาออก (หรือ alert) แทนวนเปล่า

## v6.118 — 🎯 จับ "เกจบอส" สดได้จริง — แก้บั๊ก threshold + แดงพันรอบยอด

**จับ conic-gradient ของบอสจริงได้สด (Claude-in-Chrome · logger auto-กดตีบอส ตอนบอส phase 2):**

  1. threshold `180 → 90` (ยังตัดสำเนาเล็กบน orb ~56px) + **เลือกวงแดงใหญ่สุด** (loop เก็บ `best`/`bestW` แทน return ตัวแรก → fishing เลือก 180, boss เลือก 104, กันสำเนาเล็กแย่ง)
